Investigación

URL: URL son las siglas de Uniform Resource Locator (Localizador Uniforme de Recursos en español). El URL es la dirección de todos los recursos disponibles en una red dada.

MIME: Es un estándar para formatear archivos de diferentes tipos para que puedan ser enviados a través de cualquier red.

Métodos y Códigos de errores en HTTP:

100-199: Respuestas informativas

200-299: Respuestas de éxito

300-399: Redirecciones

400-499: Errores del cliente

500-599: Errores del servidor

HTML5: Es la versión mas nueva del Hypertext Markup Language, lo cual es el código usado para sitios web. Es un conjunto de tres formas de código: HTML, CSS y JS.

JS: Es un lenguaje llamado JavaScript que permite que los sitios webs sean más que elementos estáticos que no interactúan entre sí.

Php: Es un lenguaje de script que es normalmente esta implementado en otros lenguajes.

JSP: Es una tecnología desarrollada por Java que permite la programación de sitios dinámicos y manejados por información.

Ajax: Ajax(Asynchronous JavaScript And XML) es una técnica para poder leer datos de un servidor web después de que se ha cargado una pagina permitiendo actualizaciones sin recargar una página.

Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es la red global que conecta al mundo, la WWW es la información que se puede acceder a través de él.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Usado para llamar datos de una recurso especifico

HEAD: Usado para probar que va a ser llamado por un GET.

POST: Para enviar datos al servidor con el fin de crear o actualizar un recurso. Puede ser varias veces.

PUT: Lo mismo que POST pero en nada mas una vez.

PATCH: Modifica un recurso.

DELETE: Borra un recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST, porque puede mandar la información encriptada.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se usa GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No, el código 200 significa que todo fue bien.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende, a veces el error 404 puede ocurrir porque un enlace fue escrito mal y te manda a una dirección que no existe. Otras veces porque el usuario escribió algo mal en el URL entre otros posibles casos.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Definitivamente si, el código 500 es un error general para cuando ocurre un error interno en el servidor.
