Preguntas

¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Unset destruye las variables y destroy borra la sesión. Si haces un destroy sin un unset las variables de sesión todavía van a tener valores.

¿Cuál es la diferencia entre una variable de sesión y una cookie?

La sesión esta guardada en el servidor, los cookies en el navegador del usuario.

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Si usan php puede usar una función llamada file_exists() para determinar si la existe el archivo.

¿Qué es CSRF y cómo puede prevenirse?

Es Cross Site Request Forgery. Es cuando un usuario autenticado corre codigo que otro usuario malicioso le hizo correrlo sin que ellos supieran. Esto puede ser algo innocente como forzar que el usuario se desconecte de su cuenta a quitarle dinero de su cuenta bancaria.

La forma más fácil de prevenir CSRF es tener generar un cookie cada vez que un usuario se meta a una página como una manera de autenticación.
